==========================================
PROXY ROTATION LOGIC
==========================================

Algorithm: Least Recently Used (LRU)

Selection:
1. Get all active proxies
2. Sort by: lastUsedAt ASC, useCount ASC
3. Select first proxy
4. Update lastUsedAt and increment useCount

Failure Handling:
1. If job fails, increment proxy.failCount
2. If failCount >= 3, mark proxy as "banned"
3. Retry job with different proxy

Success Handling:
1. If job succeeds, reset proxy.failCount to 0
2. Proxy remains active

==========================================
QUEUE MANAGEMENT
==========================================

States:
- Pending: Waiting to process
- Active: Currently processing
- Completed: Successfully finished
- Failed: Error occurred
- Paused: Queue is paused

Controls:
1. Pause: Stops accepting new jobs
2. Resume: Continues processing
3. Clean: Removes old jobs

Concurrency:
Default: 1 job at a time
Configurable in reports.service.ts:

limiter: { max: 2, duration: 1000 }

   - Increase delays
